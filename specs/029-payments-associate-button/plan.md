# Implementation Plan: Payments Associate Button

**Branch**: `029-payments-associate-button` | **Date**: 2026-07-02 | **Spec**: [spec.md](./spec.md)

**Input**: Feature specification from `specs/029-payments-associate-button/spec.md`

## Summary

Add an **Associate** button to the `payments/{payment_id}/` detail page that navigates operators to the payment association flow. The button is placed to the left of the existing **Edit** button, and the keyboard tab order is adjusted accordingly so the Associate button receives focus before Edit.

## Technical Context

**Language/Version**: Python 3.12, Django 5.0.14

**Primary Dependencies**: Django, HTMX (already in project via CDN in base template), Bootstrap 5, PostgreSQL

**Storage**: PostgreSQL (existing)

**Testing**: pytest + pytest-django

**Target Platform**: Web (Linux server, gunicorn)

**Project Type**: Django web application (server-rendered templates + HTMX)

**Performance Goals**: Page loads within 2 seconds; no measurable impact from adding one button

**Constraints**: Must not alter the existing payments grid, pagination, or other template structure outside the payment detail page; all user-visible strings must use i18n; button must remain left of Edit at all screen sizes

**Scale/Scope**: Single button addition on one template; no new models or database changes
