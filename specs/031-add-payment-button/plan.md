# Implementation Plan: Add Payment Button to Client Page

**Branch**: `032-add-payment-button` | **Date**: 2026-07-08 | **Spec**: [spec.md](./spec.md)

**Input**: Feature specification from `/specs/031-add-payment-button/spec.md`

## Summary

Add a **New Payment** button to the `clients/{client_id}/` detail page, positioned to the right of the existing **Nueva Reserva** button. Clicking navigates to the payment creation page (`payments/create/`) with the current client preselected in the client combo box. Direct access to `payments/create/` continues to work with an empty client field.

## Technical Context

**Language/Version**: Python 3.12, Django 5.0.14

**Primary Dependencies**: Django, HTMX, Bootstrap 5, PostgreSQL

**Storage**: PostgreSQL (existing)

**Testing**: pytest + pytest-django

**Target Platform**: Web (Linux server, gunicorn)

**Project Type**: Django web application (server-rendered templates + HTMX)

**Performance Goals**: No measurable impact from adding one link button

**Constraints**: Must not alter existing client detail page layout; all user-visible strings must use i18n; button must remain to the right of **Nueva Reserva** at all screen sizes; direct access to `payments/create/` must still work with empty client field

**Scale/Scope**: Single button addition on one template + one view modification for client preselection

## Research Plan

### Knowns (no research needed)

- **Button placement**: To the right of the existing **Nueva Reserva** button, matching the `href`-with-query-param pattern (`?client={{ client.pk }}`)
- **Target URL**: `payments:create` (maps to `PaymentCreateView`)
- **Existing pattern for client preselection**: The `PaymentCreateView.get_context_data()` already extracts `client` from `request.GET` but the form is not pre-populated — needs `get_initial()` override
- **i18n**: `"New Payment"` / `"Nuevo pago"` key already exists in `django.po`
- **Button style**: Bootstrap `btn` class, consistent with other action buttons on the page

### Unknowns (research needed)

- What Bootstrap button variant (color/size) should be used for the **New Payment** button to be visually consistent with the existing **Nueva Reserva** (`btn-primary`) and to ensure clear visual distinction from other buttons?
- Does the `PaymentForm` client field use a queryset-filters or special rendering that would affect how the preselected value is displayed?
