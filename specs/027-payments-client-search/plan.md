# Implementation Plan: Payments Client Search

**Branch**: `027-payments-client-search` | **Date**: 2026-06-29 | **Spec**: [spec.md](./spec.md)

**Input**: Feature specification from `specs/027-payments-client-search/spec.md`

## Summary

Add a client search field to the payments list page that allows operators to filter payments by client name, email, or mobile number using partial case-insensitive matching. The search uses HTMX with a 300ms debounce (matching the existing client search pattern) and preserves the existing payments grid format and pagination unchanged.

## Technical Context

**Language/Version**: Python 3.12, Django 5.0.14

**Primary Dependencies**: Django, HTMX (already in project via CDN in base template), PostgreSQL

**Storage**: PostgreSQL (existing)

**Testing**: pytest + pytest-django

**Target Platform**: Web (Linux server, gunicorn)

**Project Type**: Django web application

**Performance Goals**: Search results appear within 2 seconds of user input

**Constraints**: Must not alter payments grid template structure or pagination logic; all user-visible strings must use i18n

**Scale/Scope**: Consistent with current system — ~1,000 clients, payment volume matches existing infrastructure
