# Data Model: Translate English Tags into Spanish

## Overview

This feature does not introduce new database entities. The data involved is limited to translation entries stored in GNU gettext `.po`/`.mo` files.

## Translation Entry

| Attribute | Type | Description |
|-----------|------|-------------|
| `msgid` | string | The English source string (e.g., `"Searching..."`, `"Next"`) |
| `msgstr` | string | The Spanish translation (e.g., `"Búsqueda..."`, `"Siguiente"`) |
| `#:` comment | file path | Source file location(s) where the string appears (auto-generated by `django-admin makemessages`) |

### Translation Entries to Add

| `"Searching..."` | `"Búsqueda..."` | `clients/templates/clients/search.html` |

| `"Last"` | `"Último"` | `clients/templates/clients/_search_results.html` |
| `"Equipment"` (singular) | `"Equipo"` | `equipment/models.py` (new `verbose_name`) |

### Model Change

**Equipment model** (`backend/apps/equipment/models.py`):

Add to `Meta` class:
```python
verbose_name = _("Equipment")
```

This causes Django admin to render "Seleccione Equipo a modificar" and "AÑADIR EQUIPO" instead of the untranslated fallback.

### Existing PO Entry Conflict Resolution

The current PO file has:
```po
msgid "Equipment"
msgstr "Equipos"
```

This will be changed to:
```po
msgid "Equipment"
msgstr "Equipo"
```

This is appropriate because:
- The singular `verbose_name` requires the singular Spanish form
- The template `{% translate "Equipment" %}` works acceptably with "Equipo" as a section heading
- The plural form `msgid "equipment"` → `msgstr "equipos"` (lowercase) remains for `verbose_name_plural`
